Orchestration

Parallel Agent Fabric

三个主并行证据分支先收集事实，再汇合到 Planner、Critic 与 Policy。

Parallel Branch 01 模型：-

Change Agent

收集发布窗口、配置项变化、版本变更和可回滚目标版本。

  • 发布与告警的时间差
  • 连接串、配置包、连接池参数变化
  • 当前版本与上一稳定版本
Parallel Branch 02 模型：-

Log Agent

归纳错误模式，判断是否存在局部数据库/配置类异常。

  • 5xx 错误模式
  • 数据库连接/认证异常
  • 错误是否局限于目标服务
Parallel Branch 03 模型：-

Dependency Agent

排除下游依赖传播，把故障边界收敛到刚发布服务。

  • 下游依赖健康度
  • 是否存在错误放大
  • blast radius 是否局部可控
